USE_ASSTRING

When USE_ASSTRING applies, nl.kute.asstring.core.asString will dynamically resolve properties and values for custom classes, even if the class or any of its superclasses has toString overridden.

USE_ASSTRING is recommended unless:

  • specific requirements apply to toString methods

  • toString methods have been created with additional value, e.g. additional values for tracing objects in logs, etc.

    • NB: Such values can also be added to nl.kute.asstring.core.asString; see nl.kute.asstring.core.AsStringBuilder

Characteristics:

  • Maintenance-friendly: new or renamed properties of your custom classes are automatically reflected in the nl.kute.asstring.core.asString output

  • Full protection against errors due to recursion / self reference / mutual references (StackOverflowErrors)

  • Bypasses existing toString implementations of your custom classes

  • Fully honours annotations like nl.kute.asstring.annotation.modify.AsStringMask etc.

  • Improved String representation of lambda's, primitive arrays (like Java's int[]), Throwables etc.
